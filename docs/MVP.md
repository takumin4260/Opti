# Opti要件定義書

## 1. サービスビジョン
「家事を自動化する」ことを目的とするのではなく、「ユーザーの生活から特定の負（苦痛・時間喪失）を消滅させ、QOL（生活の質）を最大化する」ための投資判断をサポートするコンシェルジュアプリ。

---

## 2. ターゲットユーザー
- 家事の負担を減らしたいが、何から手をつければいいか分からない人。
- IoT家電の導入コストに対して、明確な納得感（ROI）を求めている人。
- 自分の住環境や生活スタイルに本当に適合する製品を知りたい人。

---

## 3. コア機能（MVP範囲）

### 3.1 ユーザー分析・診断（現状把握）
LLMが最適な提案を行うためのコンテキストを収集する。
- **住環境プロファイリング:**
    - 住居形態（賃貸/持ち家、マンション/戸建て、オートロック有無）。
    - 物理的制約（段差の有無、床材、Wi-Fi環境、家電設置スペースの有無）。
- **生活動線・タイムライン入力:**
    - 起床、外出、帰宅、就寝時間の登録。
    - 現在、各家事（掃除・洗濯・炊事等）に費やしている時間と頻度の入力。
- **ペインポイントの特定:**
    - 「嫌いな家事」だけでなく「なぜ嫌いか（腰が痛い、判断が面倒、冬の寒さが辛い等）」の理由を深掘り。
- **予算設定:**
    - 投資可能な総額、または月々の許容コストの設定。

### 3.2 LLMレコメンドエンジン
収集したデータに基づき、予算内で最適なデバイス構成を生成する。
- **最適化ロジック:**
    - 予算を優先度の高いペイン（苦痛）の解消に自動配分。
    - 単品提案だけでなく、デバイスを組み合わせた「シーン別シナリオ」での提案。
- **「負の消滅」言語化（力説機能）:**
    - 導入によって「何をしなくて良くなるか」を具体的かつ解像度高く提示。
    - 例：「帰宅時に暗い玄関で鍵を探すという無駄な3分間が、あなたの人生から消滅します」
- **製品データベース連携:**
    - 規格（Matter/HomeKit等）の互換性、設置難易度を考慮した製品抽出。

### 3.3 ROI（投資対効果）の可視化
導入の妥当性を数値と定性情報の両面から提示する。
- **時間的ROI:**
    - 年間で削減される推定時間の算出。
- **経済的ROI:**
    - ユーザーの時給換算、または電気代節約分等による投資回収期間のシミュレーション。
- **精神的ROI:**
    - 苦痛に感じていた作業からの解放度をインジケーター等で表示。

### 3.4 導入・購買サポート
- **段階的導入ロードマップ:**
    - 総予算に基づき、「今月はこれ、来月はこれ」というステップアッププランの提示。
- **購入導線:**
    - Amazon、楽天、またはレンタルサービスへのダイレクトリンク（アフィリエイト連携想定）。
